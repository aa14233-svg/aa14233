# Vol 01：C++与算力剥削

## 开章

计算的本质不是"1+1=2"，而是一条指令让CPU从内存中取出两个数，放到寄存器，执行加法，再把结果写回——全程在物理世界中移动电荷。

## 1.1 存储层次结构

| 层级 | 延迟 | 容量 | 带宽 |
|------|------|------|------|
| L1 Cache | ~1ns (3-4 cycles) | 32KB per core | ~1TB/s |
| L2 Cache | ~4ns (10-12 cycles) | 512KB per core | ~500GB/s |
| L3 Cache | ~12ns (30-40 cycles) | 16-32MB shared | ~200GB/s |
| DDR5 RAM | ~80-100ns | 32-128GB | ~50GB/s |
| NVMe SSD | ~10,000ns (10μs) | TB级 | ~7GB/s |

> 💡 **通俗解释**：Cache 就是 CPU 旁边的"高速小书桌"——你要频繁查的数据放在手边（L1），偶尔用的放稍远点的抽屉（L2），整个机房共享的公共书架（L3）。如果真的去仓库（内存）拿，要等 80-100 个周期，到硬盘去找更是要等上万年（相对CPU周期而言）。所有AI推理优化，本质就是让数据尽可能留在手边。

**对Agent推理的意义**：每一轮Agent调用背后，Transformer都在执行大规模矩阵乘法。推理引擎（vLLM、llama.cpp）的全部优化，本质是**最大化L1/L2命中率**。

> **🧑‍💻 TA说**  
> 我在实际做ncnn推理优化时，最典型的优化就是把for循环从i→j遍历改为j→i遍历——仅仅改变访问顺序，推理速度能差3-5倍。这就是缓存行效应在工程中的具象化。

## 1.2 矩阵乘法优化：Loop Tiling

朴素三层循环的问题：内层对B的访问是列主序，每次缓存缺失。

> 💡 **通俗解释**：想象你在整理一个巨型书架上的书，一次只能抱一摞（一摞≈Cache容量）。朴素三层循环相当于每次从书架A拿一本，跑到书架B找对应的书，再跑回来——来回跑废腿。Tiling 就是"一次性把相邻一摞书都抱到桌上，在桌上做完所有运算再放回去"。桌上操作比来回跑快几十倍。

Tiling核心思想：将大矩阵分块，每块恰好装入L1/L2缓存。

## 1.3 SIMD：一次指令处理一组数据

AVX-512允许一条指令完成16个float乘法。配合FMA，单周期32次浮点运算。

> 💡 **通俗解释**：普通指令像"逐个学生发卷子"——一次发一张。SIMD像"把一排16个学生叫上讲台，同时发卷子"。AVX-512 就是那种能同时处理16个人的指令。FMA（融合乘加）更进一步：发卷子的同时顺带收作业——一条指令干了两件事。

> **🧑‍💻 TA说**  
> 微软的DirectML和Intel的oneDNN底层全是这样手写的SIMD kernel。PyTorch的aten算子最终编译出来也是这些指令。理解SIMD就是在理解你用的框架最后一公里怎么跑的。

## 章节小结

无论你写的是Agent框架还是推理引擎，最终都要落到"数据在哪儿、如何搬动它"。Loop Tiling和SIMD是两把最基础的钥匙。
