# Vol 03：Embedding映射

## 开章

"猫"和"狗"在语义空间中很近。Embedding要解决的核心问题：**把离散符号投射到连续向量空间，让语义相似度 ≈ 向量距离**。

> 💡 **通俗解释**：Embedding = 给每个词一个"GPS坐标"。Apple(水果)和Banana的坐标很近，Apple(苹果公司)和Samsung的坐标很近。坐标是几百维的（如768维），人脑想象不了，但计算机可以。语义相似 = 坐标距离小。

## 3.1 向量空间模型

```
"国王" - "男人" + "女人" ≈ "女王"
similarity(w₁, w₂) = cos(e₁, e₂) = (e₁·e₂) / (‖e₁‖·‖e₂‖)
```

→ 这是[[教材：The Automated Mind/Phase 2：语义空间与认知具象/Vol 04：RAG与知识图谱|Vol 04向量检索]]的数学基础。

> **🧑‍💻 TA说**  
> 经典的"国王-男人+女人=女王"向量运算不是童话。我在实际项目里用这个特性做概念类比搜索——比如输入"iPhone→App Store"找到"Android→Google Play"。这比关键词匹配灵活得多。

## 3.2 余弦相似度的SIMD优化

1024维余弦相似度：AVX-512每周期16维，64周期完成 → [[教材：The Automated Mind/Phase 1：秩序的底座/Vol 01：C++与算力剥削|Vol 01 SIMD]]的直接应用。

> **🧑‍💻 TA说**  
> 实际做语义搜索时，Cosine相似度比欧氏距离更常用。原因是embedding的"方向"比"长度"携带更多语义信息——就像我们判断两个人观点是否一致，看方向（立场）比看距离（表达强度）更有意义。

## 3.3 维度灾难的本质

高维空间中几乎所有点都"一样远"→ 基于距离的相似度搜索会失效。

> 💡 **通俗解释**：在2D房间里，两个点之间的距离差别明显（最近的1米，最远的100米）。但在1000维的"超级房间"里，所有点之间的距离几乎相等——最近的95米，最远的100米。这就像所有人站在足球场上却感觉彼此相距一样远，距离度量就失去了区分意义。这就是为什么高维Embedding需要先降维再用ANN索引。

→ 这正是[[教材：The Automated Mind/Phase 4：深水区与几何/Vol 06：流形与几何学|Vol 06流形假设]]要回答的问题。

## 章节小结

Embedding是连接离散世界和连续世界的桥梁。理解它才能理解为什么RAG需要ANN索引、为什么维度灾难是检索天花板。
