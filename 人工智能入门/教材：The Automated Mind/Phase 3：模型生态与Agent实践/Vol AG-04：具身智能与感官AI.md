# Vol AG-04：具身智能与感官AI

## 开章

到目前为止，我们讨论的Agent都在"屏幕里"——处理文本、图像、声音这些数字信号。但真实世界的智能需要**走出屏幕**：控制机械臂抓取物体、驱动无人机避开障碍、在智能家居中根据语音命令调节环境。

当推理能力从云端部署到物理设备时，约束条件发生质变：

| 约束 | 云端Agent | 具身/感官Agent |
|------|-----------|---------------|
| 延迟容忍 | 秒级 | **毫秒级**（机器人控制） |
| 网络依赖 | 在线 | **可能断网** |
| 模型大小 | 无限制 | **<1GB**（端侧） |
| 功耗 | 不限 | **电池约束** |
| 传感器输入 | 文本/文件 | **摄像头/麦克风/IMU/LiDAR** |

这一卷讲两件事：如何让Agent与物理世界交互（具身智能），以及如何让Agent"看"和"听"（感官AI）。

---

## 4.1 具身智能概述

### 典型架构

```
传感器（摄像头 / LiDAR / 麦克风 / 触觉）
  ↓
端侧推理模型（量化后 < 500MB）
  ↓
状态机 / 强化学习策略（动作规划）
  ↓
执行器（PWM信号 / 电机控制 / GPIO）
  ↓
物理环境 → 传感器 → 闭环
```

### 与云端Agent的关键区别

| 维度 | 云端Agent | 具身Agent |
|------|-----------|-----------|
| 感知 | API输入 | 传感器实时数据流 |
| 推理 | 大模型，可长时间思考 | 小模型，必须在**控制周期内**完成 |
| 行动 | 返回文本/调用API | **物理动作**——不可撤销 |
| 安全 | 输出错误可retry | 输出错误可导致物理损坏 |

### 具身Agent的部署优化

具身Agent的部署是"用有限的计算资源完成实时决策"——这是[[教材：The Automated Mind/Phase 1：秩序的底座/Vol 01：C++与算力剥削|Vol 01算力剥削]]和[[教材：The Automated Mind/Phase 3：模型生态与Agent实践/Vol 08：Agent部署|Vol 08 FSM]]的实战交汇点。

**优化手段**：
- **模型量化**：FP16→INT8→INT4，精度损失<2%但显存减半
- **知识蒸馏**：大模型→小模型，保留核心能力
- **剪枝**：移除"不重要"的神经元/注意力头
- **算子融合**：多个小op合并为一个大op，减少Kernel Launch开销

## 4.2 端侧部署专项

### 端侧模型选型

| 设备类型 | 可用内存 | 推荐模型 | 量化方式 |
|----------|---------|----------|---------|
| 手机 | 4-8GB | Qwen2.5-0.5B~3B / Gemma 2B | GGUF INT4 |
| 树莓派/边缘盒子 | 2-4GB | Phi-2 / TinyLlama 1.1B | GGUF INT4 |
| 智能家居Hub | 1-2GB | MobileNet / Whisper tiny | TFLite INT8 |
| 微控制器(ESP32) | <1MB | 纯规则+朴素贝叶斯 | 无模型 |

### 部署框架

| 框架 | 适用平台 | 特点 |
|------|---------|------|
| llama.cpp | CPU / ARM | GGUF量化、C++底层、推理最快 |
| TensorFlow Lite | 移动端 / 嵌入式 | Google生态、硬加速支持 |
| ONNX Runtime | 跨平台 | 微软生态、算子兼容性最好 |
| ExecuTorch | Apple设备 | 苹果神经网络引擎加速 |
| MediaPipe | 移动端 | Google、重点是视觉管线和手势识别 |

## 4.3 视觉模型实战

### 任务与模型矩阵

| 任务 | 推荐模型 | 推理方式 | 端侧可行性 |
|------|----------|----------|-----------|
| **图像分类** | CLIP / SigLIP | 一次前向 | ✅ TinyCLIP可跑在手机 |
| **目标检测** | YOLOv8n / RT-DETR | 一次前向输出BBox | ✅ YOLOv8n ~ 3MB |
| **语义分割** | SAM 2 | Prompt-guided | ⚠️ 需云端辅助 |
| **姿态估计** | MMPose / MoveNet | 关键点回归 | ✅ 实时姿态 |
| **OCR** | PaddleOCR / TrOCR | 检测+识别流水线 | ⚠️ 需优化 |
| **图像生成** | SDXL / FLUX | 迭代去噪 | ❌ 需云端 |

### Agent集成示例：视觉质检Agent

```
产品照片流入
  → YOLOv8 检测缺陷区域（毫秒级）
  → SAM 2 分割缺陷轮廓（可配置精度）
  → LLM 评估缺陷严重等级 + 生成质检报告
  → 触发生产线报警 / 标记剔除
```

## 4.4 音频模型实战

### 任务与模型矩阵

| 任务 | 推荐模型 | 关键指标 | 端侧可行性 |
|------|----------|----------|-----------|
| **语音识别(ASR)** | Whisper large-v3 / SenseVoice | WER <3%（英文） | ✅ Whisper tiny |
| **语音合成(TTS)** | CosyVoice / Bark / Coqui-AI | MOS >4.0 | ⚠️ 需够量优化 |
| **说话人识别** | ECAPA-TDNN / ResNetSE | EER <2% | ✅ 轻量版 |
| **音乐生成** | MusicGen / Suno API | 主观质量 | ❌ 需云端 |
| **声音事件检测** | YAMNet / BEATS | 527类音频事件 | ✅ 手机实时 |

### Agent集成示例：智能会议纪要Agent

```
实时会议音频流
  → Whisper 实时转写（streaming ASR）
  → LLM 语义分割（话题切换检测）
  → 逐话题生成摘要
  → 提取待办事项 + 责任人
  → 自动发送纪要邮件
```

### 关键技术：语音活动检测（VAD）

VAD是音频Agent的入口——决定"什么时候有人在说话"。推荐Silero VAD，在手机端延迟<10ms，准确率>95%。

## 4.5 数据库管理Agent

### NL2SQL（自然语言→SQL）

```
用户问题："上个月销售额最高的TOP5产品是什么？"

Agent执行流程：
1. Schema理解 → 读取表结构+字段注释
2. 问题解析 → 识别"上个月"=时间过滤、"销售额最高"=SUM+ORDER BY
3. SQL生成 → 生成候选SQL并做语法校验
4. 查询执行 → 连接数据库执行
5. 结果解释 → 将查询结果转自然语言
```

### 数据库管理Agent能力矩阵

| 任务 | 难度 | Agent成熟度 | 代表工具 |
|------|------|-----------|---------|
| SQL生成 | 中 | ⭐⭐⭐⭐ | Text2SQL / DAIL-SQL |
| Schema迁移 | 高 | ⭐⭐ | 需人工审核 |
| 数据清洗 | 中 | ⭐⭐⭐⭐ | 规则+LLM联合 |
| 性能诊断 | 高 | ⭐⭐ | 需DBA经验积累 |
| 异常检测 | 中 | ⭐⭐⭐⭐ | 时序+LLM |

## 4.6 其他补充：传感器融合与环境智能

### 传感器数据融合Agent

典型应用：智能温室、工业IoT、智能家居

```
多传感器输入（温度/湿度/光照/CO₂/土壤湿度）
  → 时序数据预处理（归一化+异常过滤）
  → 轻量模型预测最优控制策略
  → 执行器控制（灌溉/通风/补光）
  → 长期优化（基于产量/能耗的反馈闭环）
```

### 生物信号处理（初步）

Agent也开始进入生物信号领域：
- **脑机接口（BCI）**：EEG信号分类 → 意念控制
- **心电监测（ECG）**：异常心律检测 → 实时预警
- **肌电信号（EMG）**：手势识别 → 假肢控制

目前成熟度有限（⭐⭐），但方向明确——Agent正在从"处理文字"扩展到"处理任何信号"。

## 章节小结

具身智能将Agent从数字世界带入物理世界，感官AI让Agent拥有了"眼睛"和"耳朵"。这两个领域是Agent落地下一个爆发点。技术关键不是模型本身的高级程度，而是**在算力、功耗、延迟的三重约束下做出有效决策**——这正是全书从[[教材：The Automated Mind/Phase 1：秩序的底座/Vol 01：C++与算力剥削|Vol 01]]就开始铺垫的核心能力。
