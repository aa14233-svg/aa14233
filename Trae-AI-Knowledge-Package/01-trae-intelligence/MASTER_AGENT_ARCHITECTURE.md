# Trae 主调度 Agent 架构

## 主调度 Agent 名称
**TraeOrchestrator** — 核心调度和监工 Agent

## 一、核心职责
1. **任务路由和分发** — 接收用户任务 → 技能识别 → 路由给对应 Agent
2. **Hook + Loop Engineering 约束** — 在关键节点插入约束钩子
3. **流程化验收** — D1(阶段)/D2(模块)/D3(总体)
4. **全程监工** — 实时监控、负载均衡、异常处理

## 二、Hook + Loop Engineering

| Hook 阶段 | 触发时机 | 约束检查 |
|-----------|---------|---------|
| 任务准备 Hook | 分发前 | 安全、资源、依赖 |
| 执行中 Hook | 每1小阶段 | 进度、质量、风险 |
| 验收 Hook | 子/全任务完成 | 验收标准、缺陷 |
| 完成 Hook | 交付前 | 最终检查、文档 |

```
1. 任务分解 → 子任务列表
2. 分发 → 指定 Agent
3. 执行 Hook 检查 → 继续/调整/回滚
4. 子阶段验收 → 合格继续，不合格反馈
5. 循环直到所有子任务完成
6. 最终 Hook + 总体验收
```

## 三、验收等级
- **D1** (阶段验收) — 单个小任务完成
- **D2** (模块验收) — 相关功能模块完成
- **D3** (总体验收) — 全任务完成

### 验收检查清单
- [ ] 功能完整性
- [ ] 代码/交付质量
- [ ] 文档完整性
- [ ] 用户需求匹配度
- [ ] 安全约束遵守

## 四、监工指标
- Agent 健康度、任务进度、资源占用、执行时间、成功率

## 五、失败处理
- 单个 Agent 失败 → 自动降级备选 Agent
- 验收不通过 → 反馈 → Agent 修正 → 重新验收
- 资源不足 → 调整优先级 → 队列管理
